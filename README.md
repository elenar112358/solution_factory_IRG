# solution_factory_IRG
Тестовое задание для проекта "Фабрика решений"

**GET /obligations**  
По условию непонятно, условия на category и status как ***AND***, или ***OR***.  
Реализовано ***AND***.  

***Бизнес-правило (lazy expiry). Перед формированием ответа сервис переводит все записи со статусом
active и next_payment_date < today в статус expired.
Рекуррентные обязательства (recurrence != null) под это
правило не попадают: если пользователь не нажал «Оплачено», дата просто устарела, но подписка не
прекратилась. Для них статус остаётся active.***  

Обоснование:  
Рекуррентные обязательства всегда активны, даже устаревшие. 
Они не могут быть expired, по ним задолженность будет накапливаться до отмены.
Это необходимо учитывать при расчете текущей задолженности пользователя по обязательствам.
Задолженность погасится в базе только при внесении оплаты - и будет сдвинута дата.
По факту по ней уже прошло списание денежных средств с карты (при их наличии) 
или произойдет, когда на карте что-то появится. 
В любом случае, эта ситуация должна вручную контролироваться пользователем.
